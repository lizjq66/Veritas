This directory contains the artifact of Veritas's most recent
deterministic verification session. The text artifacts are
auto-generated by `pytest tests/test_loop.py` and committed so anyone
can see what the verifier did without running it.

- `events.jsonl` — step-by-step event stream from the example runner.
- `summary.md` — human-readable verification report, including a
  fixture batch that shows explicit Gate 1 / Gate 2 / Gate 3 verdicts.
- `journal.db` — SQLite journal of every proposal that cleared all
  three gates in the session, plus the post-approval classification
  once the caller's executor closed out (open with `sqlite3`).
  **Not committed** — its rows carry per-run timestamps, so the binary
  always shows dirty even when the verdicts are byte-for-byte
  reproducible (the test asserts a timestamp-free row hash). Run the
  test above to regenerate it locally; the committed `events.jsonl` /
  `summary.md` capture the same session in diffable text.

Note: market data here is synthesized by the test fixtures in
`test_loop.py`. What is real is the verifier — the Lean kernel
behind each gate, the reliability update rule, and the deterministic
reproducibility of the verdicts. Swapping the example observer and
executor (`python/observer.py`, `python/executor.py`) for a real
venue adapter does not change anything the verifier does.
